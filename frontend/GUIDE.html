Getting Started with EduNova

A complete step-by-step guide to launch the platform locally — from zero to running in under 10 minutes.

✅ No prior setup needed ⚡ 10-minute quickstart 🎓 Full-stack platform

🏗 Platform overview

🌐
Frontend
Pure HTML + CSS + JS. No framework, no build step. Open index.html in a browser.
⚙️
Backend
NestJS REST API with TypeORM. Requires Node.js 20+ and MySQL 8.0+.
🗄️
Database
MySQL 8.0 with full schema, seeds, indexes, views. Import via phpMyAdmin or CLI.

🌐 Launch the Frontend
The frontend is 100% static — no Node.js, no build process needed. Three ways to open it:

Option A — Open directly (simplest)

# On macOS open frontend/index.html # On Windows start frontend/index.html # On Linux xdg-open frontend/index.html

Option B — Serve with npx (recommended for links to work)

# Install serve globally (one time) npm install -g serve # Serve the frontend folder serve frontend/ # Or with npx (no install needed) npx serve frontend/ # Output: → Local: http://localhost:3000 → Network: http://192.168.x.x:3000

Option C — VS Code Live Server

💡
Install the Live Server extension in VS Code. Then right-click frontend/index.html → Open with Live Server. It auto-reloads on changes.
✅
Frontend works standalone — all pages are fully functional with simulated data. You can demo the entire UI without setting up the backend or database.

🗄️ Setup the Database
Import the MySQL schema into a new database. Choose phpMyAdmin (visual) or CLI (faster).

Option A — phpMyAdmin (visual, beginner-friendly)

  1. Open http://localhost/phpmyadmin
  2. Click "New" in the left sidebar
  3. Database name: edunova_db
  4. Collation: utf8mb4_unicode_ci → Create
  5. Click the "Import" tab
  6. Choose file: database/schema.sql
  7. Click "Go" → Done! ✅

Option B — MySQL CLI (faster)

# Login to MySQL mysql -u root -p # Inside MySQL shell — create database CREATE DATABASE edunova_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; EXIT; # Import schema mysql -u root -p edunova_db < database/schema.sql
🌱
The schema automatically seeds: 8 categories, 9 gamification badges, and a default admin account (admin@edunova.com).
⚠️
Make sure MySQL 8.0+ is running. XAMPP and WAMP include phpMyAdmin + MySQL. On macOS, use brew install mysql or use MAMP.

⚙️ Launch the Backend API
The NestJS backend powers all data, authentication, and business logic. Requires Node.js 20+.
# 1. Check Node.js version (need 20+) node --version # Should output: v20.x.x or higher # 2. Navigate to backend folder cd backend # 3. Install all dependencies npm install # 4. Copy environment file cp .env.example .env # Then edit .env with your DB credentials (see Step 4) # 5. Start in development mode (with hot reload) npm run start:dev # You'll see: 🚀 EduNova API running on: http://localhost:3000/api/v1 📚 Swagger docs: http://localhost:3000/api/docs
📦
npm install downloads all dependencies including NestJS, TypeORM, bcrypt, JWT, Swagger, etc. Takes ~1-2 minutes. Requires internet access.

🔧 Configure Environment Variables
Create a .env file in the backend/ folder with these settings:
# ── App ─────────────────────────────── NODE_ENV=development PORT=3000 CORS_ORIGINS=http://localhost:3001,http://127.0.0.1:5500 # ── Database ─────────────────────────── DB_HOST=localhost DB_PORT=3306 DB_USERNAME=root DB_PASSWORD=your_mysql_password DB_NAME=edunova_db # ── JWT Tokens ───────────────────────── JWT_SECRET=edunova-super-secret-change-me-2026 JWT_EXPIRES_IN=15m JWT_REFRESH_SECRET=edunova-refresh-secret-change-me JWT_REFRESH_EXPIRES=7d # ── Email (Gmail SMTP) ───────────────── MAIL_HOST=smtp.gmail.com MAIL_PORT=587 MAIL_USER=your.email@gmail.com MAIL_PASS=your_gmail_app_password MAIL_FROM="EduNova <noreply@edunova.com>" # ── File Uploads ─────────────────────── UPLOAD_PATH=./uploads MAX_FILE_SIZE=104857600 CDN_URL=http://localhost:3000/uploads
⚠️
For Gmail, enable 2-Step Verification then create an App Password at myaccount.google.com/apppasswords. Use that as MAIL_PASS.
🔒
Never commit .env to Git. It's already listed in .gitignore. Change all default secrets before production deployment.

👤 User Roles & Demo Access
EduNova has three separate roles. The frontend demo works without backend — just use any credentials:
🧑‍💻
Candidate
Learner role. Browse courses, enroll, watch lessons, take quizzes, earn certificates.
→ pages/candidat/dashboard.html
👩‍🏫
Trainer
Creator role. Build courses, upload videos, manage students, view revenue analytics.
→ pages/formateur/dashboard.html
🔒
Admin
Platform manager. Validate trainers/courses, view all analytics, manage support.
→ pages/admin/dashboard.html

Demo Login Flow (frontend only):

  1. Open pages/login.html
  2. Select Candidate or Trainer role card
  3. Enter any email + password (e.g. test@test.com / password123)
  4. Click Sign in → redirected to the correct dashboard
  5. For Admin: click the Admin portal link at the bottom of login page

📡 API Quick Reference
Key endpoints once the backend is running. All protected endpoints require Authorization: Bearer <token>.
# Test API health curl http://localhost:3000/api/v1 # Register a new candidate curl -X POST http://localhost:3000/api/v1/auth/register \ -H "Content-Type: application/json" \ -d '{"prenom":"Sara","nom":"Aissaoui","email":"sara@test.com","motDePasse":"Pass1234!","role":"CANDIDAT"}' # Login and get JWT curl -X POST http://localhost:3000/api/v1/auth/login \ -H "Content-Type: application/json" \ -d '{"email":"sara@test.com","motDePasse":"Pass1234!"}' # Get published courses curl http://localhost:3000/api/v1/cours # Get courses with filters curl "http://localhost:3000/api/v1/cours?niveau=DEBUTANT&page=1&limit=12" # Verify a certificate (public endpoint) curl http://localhost:3000/api/v1/certificats/verifier/EDN-2026-XYZ-00001
📚
The full interactive API docs are at http://localhost:3000/api/docs. You can test every endpoint directly from Swagger — no Postman needed.
8
🐳 Docker (One-Command Setup)
Run the entire stack (MySQL + NestJS API) with a single command using Docker Compose.

Create docker-compose.yml in the project root:

version: '3.8' services: db: image: mysql:8.0 environment: MYSQL_ROOT_PASSWORD: rootpass MYSQL_DATABASE: edunova_db MYSQL_CHARACTER_SET_SERVER: utf8mb4 ports: - "3306:3306" volumes: - mysql_data:/var/lib/mysql - ./database/schema.sql:/docker-entrypoint-initdb.d/schema.sql api: build: ./backend ports: - "3000:3000" environment: DB_HOST: db DB_PORT: 3306 DB_USERNAME: root DB_PASSWORD: rootpass DB_NAME: edunova_db JWT_SECRET: edunova-docker-secret depends_on: - db volumes: mysql_data:
# Build and start all services docker-compose up -d # Check running services docker-compose ps # View logs docker-compose logs -f api # Stop everything docker-compose down

🔧 Troubleshooting
Common issues and their solutions:
❌ "Cannot connect to MySQL"
Make sure MySQL is running. Check with mysqladmin -u root -p status. Verify DB_HOST, DB_PORT, DB_USERNAME, DB_PASSWORD in your .env file match your MySQL installation.
❌ "Port 3000 already in use"
Change the port in .env: PORT=3001. Or kill the process using the port: lsof -ti:3000 | xargs kill (macOS/Linux).
❌ "Node version too old"
Install Node.js 20+ from nodejs.org or use nvm: nvm install 20 && nvm use 20.
❌ Frontend links not working (404)
Open the files through a server (use npx serve frontend/) instead of double-clicking. File:// protocol doesn't resolve relative paths for navigation correctly.
❌ "Module not found" after npm install
Delete node_modules and package-lock.json then run npm install again. Make sure you are inside the backend/ folder.
❌ CORS error when calling API
Add your frontend URL to CORS_ORIGINS in .env. Example: CORS_ORIGINS=http://localhost:3001,http://127.0.0.1:5500.
